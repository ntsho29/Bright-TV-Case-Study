# Bright-TV-Case-Study

## 📖 Project Overview

This project focuses on analysing Bright TV user and viewership data using SQL in Databricks. The project involved cleaning, transforming and preparing raw data for analysis by applying various SQL techniques. The objective was to produce reliable, well-structured data that can be used to answer business questions and support data-driven decision-making.
---

## 🎯 Project Objectives

- Clean and prepare raw datasets for analysis.
- Identify and remove duplicate records.
- Standardise inconsistent data values using SQL.
- Create meaningful categories using CASE statements.
- Extract useful information from date and time fields.
- Combine multiple datasets using SQL joins.
- Produce a clean dataset suitable for reporting and analysis.
---

## 🔑 Key Findings

During the project, the data was successfully cleaned and transformed into a more structured format. This included:

- Identifying duplicate records.
- Handling missing and inconsistent values.
- Categorising users into meaningful age groups.
- Classifying viewing behaviour by weekday/weekend and time of day.
- Combining user profile data with viewership data to create a richer dataset for analysis.
---
## 💡 What I Learned

This project helped me understand that data analysis is not just about writing SQL queries, but about solving business problems by cleaning, transforming and interpreting data to produce meaningful insights. Throughout this project, I learned how to clean and prepare real-world datasets for analysis, combine information from multiple tables using SQL joins, apply aggregate functions to summarise data, create calculated fields using CASE statements, handle missing values using COALESCE, and use date functions to analyse trends over time. Most importantly, I developed the ability to write cleaner, more structured and readable SQL queries while thinking from a business perspective.
